# 数据采集频率说明

## ❓ 问题：为什么不能1分钟采集一次？

### 原始想法（错误的）
修改数据采集频率从5分钟改为1分钟，以确保不错过早盘数据。

### 实际问题 ⚠️

**关键代码**：
```java
// 每1分钟执行一次（错误）
@Scheduled(fixedRate = 60000)
public void collectMarketData() {
    // 获取5分钟K线
    JsonArray klines = bybitTradingService.getKlines(bybitSymbol, "5", 1);
}
```

**问题分析**：
1. **K线周期是5分钟**：`"5"`表示5分钟K线
2. **5分钟K线每5分钟才更新一次**
3. **如果每1分钟采集**：
   - 第1分钟：获取K线 [10:00-10:05, close=4565.30]
   - 第2分钟：获取K线 [10:00-10:05, close=4565.50] ← 同一根K线
   - 第3分钟：获取K线 [10:00-10:05, close=4565.80] ← 同一根K线
   - 第4分钟：获取K线 [10:00-10:05, close=4566.00] ← 同一根K线
   - 第5分钟：获取K线 [10:00-10:05, close=4566.20] ← 同一根K线
   - 第6分钟：获取K线 [10:05-10:10, close=4567.00] ← 新的K线

4. **导致的问题**：
   - 数据库中存储大量重复的K线（只是close价格略有变化）
   - EMA计算需要50根K线，但实际只有10根真实K线（其他40根是重复的）
   - **指标计算会严重失真**
   - 数据库空间浪费

## ✅ 正确的做法

### 方案1: 保持5分钟采集（当前使用）⭐⭐⭐⭐⭐
```java
@Scheduled(fixedRate = 300000)  // 5分钟
public void collectMarketData() {
    // 获取5分钟K线
    JsonArray klines = bybitTradingService.getKlines(bybitSymbol, "5", 1);
}
```

**优点**：
- ✅ 采集频率与K线周期匹配
- ✅ 不会有重复数据
- ✅ 指标计算准确
- ✅ 数据库不会膨胀

**缺点**：
- ⚠️ 如果系统崩溃或重启，可能错过某些K线

### 方案2: 使用1分钟K线（需要大改）
```java
@Scheduled(fixedRate = 60000)  // 1分钟
public void collectMarketData() {
    // 获取1分钟K线
    JsonArray klines = bybitTradingService.getKlines(bybitSymbol, "1", 1);
}
```

**优点**：
- ✅ 数据更细粒度
- ✅ 不会错过早盘数据

**缺点**：
- ❌ 需要修改整个系统（所有策略都是基于5分钟K线）
- ❌ 需要重新校准所有指标参数
- ❌ 数据量增加5倍（1分钟vs5分钟）
- ❌ 计算量增加

### 方案3: 启动时补全历史数据 ⭐⭐⭐⭐
**推荐方案**：系统启动时，自动从Bybit拉取历史K线数据

```java
@PostConstruct
public void initHistoricalData() {
    // 启动时拉取最近1000根5分钟K线
    JsonArray klines = bybitTradingService.getKlines(bybitSymbol, "5", 1000);
    // 批量保存到数据库
    saveKlineBatch(klines);
}
```

**优点**：
- ✅ 确保有足够的历史数据
- ✅ 即使错过早盘，也能补全数据
- ✅ EMA、MACD等指标计算准确
- ✅ 不会有重复数据

## 🔍 今日数据缺失原因

查看今日K线数据：
```
07:20:00 - 有数据
10:00:00 - 有数据
```

**数据是完整的！**不是采集频率的问题。

真正的问题是：**系统10:49才开始执行策略**，而不是数据缺失。

## 💡 真正的解决方案

### 问题：错过早盘交易
**原因**：策略执行时间太晚（10:49才开始）
**解决**：历史数据是存在的，只需要EMA趋势识别即可

### 系统工作流程（修复后）

1. **7:20**: K线数据存在（从数据库查询）
2. **8:00**: K线数据存在
3. **10:00**: 策略执行
   - 查询最近100根K线（包含7:20-10:00的数据）
   - 计算EMA20/EMA50
   - 识别趋势：上涨
   - 生成做多信号
   - 执行交易 ✅

**关键**：
- ✅ 不需要改采集频率
- ✅ 数据已经存在数据库中
- ✅ 策略会读取历史数据进行分析
- ✅ EMA会基于过去50根K线计算

## 📊 数据流程图

```
Bybit服务器
    ↓ (每5分钟)
collectMarketData()
    ↓
保存到数据库
    ↓ (每10秒)
executeStrategy()
    ↓
查询最近100根K线（包含历史）
    ↓
计算EMA20/EMA50
    ↓
识别趋势（基于历史数据）
    ↓
生成交易信号
```

## ✅ 最终结论

1. **保持5分钟采集频率**（已恢复）
2. **历史数据已经存在**（从7:20开始）
3. **EMA会读取历史数据**（查询100根K线）
4. **不会错过早盘趋势**（历史数据包含早盘）

**核心改进是EMA趋势识别，而不是采集频率！**
