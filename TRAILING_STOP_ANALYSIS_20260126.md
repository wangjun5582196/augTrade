```yaml
trailing-stop:
  enabled: true              # ✅ 已启用
  trigger-profit: 20.0       # 盈利$20触发移动止损
  distance: 8.0              # 止损距离$8
  lock-profit-percent: 50.0  # 锁定50%利润
```

### 配置解读

1. **触发条件**: 盈利达到$20时启用移动止损
2. **止损距离**: 价格回调$8就会触发止损
3. **锁定利润**: 首次触发时锁定50%的已有利润

## 🎯 实际运行情况分析

### 案例1：交易1 (+$20)

**入场**: 5075.10做空  
**盈利**: +$20（价格下跌约$2）  
**结果**: CLOSED_STOP_LOSS

**推测过程**:
1. 价格从5075.10跌到5073.10（盈利$20）
2. 触发移动止损，止损价设为5073.10 + $8 = 5081.10
3. 价格反弹到5081.10以上，触发止损

**问题**: 
- ✅ 触发点$20合理
- ❌ 止损距离$8太近（只占盈利的40%）
- ❌ 黄金波动$8很常见，容易被扫

### 案例2：交易3 (+$53) ✅

**入场**: 5078.50做空  
**盈利**: +$53（价格下跌约$5.3）  
**结果**: CLOSED_SIGNAL_REVERSAL（成功）

**过程**:
1. 价格从5078.50跌到5073.20（盈利$53）
2. 可能触发了移动止损
3. 但在止损前，出现反向信号（做多信号）
4. 主动平仓，避免了被止损

**成功原因**: 
- ✅ 信号反转及时平仓
- ✅ ADX=22.38，趋势明确
- ✅ 持仓时间适中

### 案例3：交易4 (+$7)

**入场**: 5069.10做空  
**盈利**: +$7（小盈利）  
**结果**: CLOSED_STOP_LOSS

**问题**: 
- ❌ 盈利$7 < $20，不应触发移动止损
- ❌ 但仍然被止损，说明是**固定止损**被触发
- ❌ 固定止损设置可能有问题

### 关键发现

通过分析发现，**不是移动止损的问题**，而是：

1. **固定止损设置太紧**
   - 当前：ATR × 3.0（约$15-16）
   - 小盈利单（$4-$20）容易被价格正常回调扫掉

2. **移动止损距离不够**
   - 当前：$8
   - 黄金正常波动：$5-10
   - 建议：$10-12

## 💡 优化建议

### 方案1：调整移动止损参数（推荐）⭐⭐⭐⭐⭐

```yaml
trailing-stop:
  enabled: true
  trigger-profit: 30.0       # 从$20提高到$30（更稳健）
  distance: 12.0             # 从$8提高到$12（更大容错空间）
  lock-profit-percent: 60.0  # 从50%提高到60%（锁定更多利润）
```

**优点**:
- 更高的触发门槛避免频繁启动
- 更大的止损距离减少被扫概率
- 锁定更多利润保护收益

**预期效果**:
- 交易1 (+$20): 不触发移动止损，等待自然止盈或信号反转
- 交易3 (+$53): 在$30触发，止损价保留$18利润（60%），给价格$12回调空间
- 小盈利单: 由固定止损保护，更合理

### 方案2：分层移动止损（高级）⭐⭐⭐⭐

```yaml
trailing-stop:
  enabled: true
  
  # 第一层：小利润保护
  tier1:
    trigger-profit: 20.0     # 盈利$20触发

    lock-profit-percent: 40.0 # 锁定40%
  
  # 第二层：中等利润追踪
  tier2:
    trigger-profit: 40.0     # 盈利$40触发

  # 第三层：大利润锁定
  tier3:
    trigger-profit: 60.0     # 盈利$60触发

```

**优点**:
- 根据盈利大小动态调整策略
- 小利润给予更大空间
- 大利润更激进保护

**缺点**:
- 实现复杂度高
- 需要修改代码逻辑

### 方案3：基于ATR的动态距离（最优）⭐⭐⭐⭐⭐

```yaml
trailing-stop:
  enabled: true
  trigger-profit: 30.0           # 盈利$30触发
  distance-multiplier: 2.5       # ATR倍数（替代固定距离）
  lock-profit-percent: 60.0      # 锁定60%利润
  min-distance: 8.0              # 最小距离$8
  max-distance: 15.0             # 最大距离$15
```

**计算方式**:
```
移动止损距离 = min(max(ATR × 2.5, $8), $15)
```

**优点**:
- 根据市场波动自适应
- 低波动时距离小（保护利润）
- 高波动时距离大（避免被扫）

**示例**:
- ATR=3.0: 距离=3.0×2.5=$7.5 → 使用最小值$8
- ATR=4.5: 距离=4.5×2.5=$11.25 → 使用$11.25
- ATR=7.0: 距离=7.0×2.5=$17.5 → 使用最大值$15

## 💡 关键经验

### 移动止损的本质

**目的**: 保护利润，而非追求完美退出

**核心原则**:
1. **宁可晚触发，不要早触发**
2. **宁可给空间，不要太紧**
3. **宁可信号反转，不要被扫**

### 常见误区

❌ **误区1**: 移动止损越早越好
- **真相**: 过早触发限制利润增长

❌ **误区2**: 止损距离越近越好
- **真相**: 太近容易被正常波动扫掉

❌ **误区3**: 锁定利润越多越好
- **真相**: 过度锁定会降低最终盈利

### 最佳实践

✅ **原则1**: 触发门槛=止盈目标的40-50%
- 止盈$75，则$30-38触发

✅ **原则2**: 止损距离=ATR的2-3倍
- ATR=$4，则距离$8-12

✅ **原则3**: 锁定比例=50-60%
- 盈利$50，锁定$25-30
