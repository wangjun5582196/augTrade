# 补齐K线数据

按每天288根（5分钟K线，24h×12=288）为标准，自动检测缺口并通过OKX接口补齐。

## 执行步骤

### 第一步：查询缺口日期

执行以下SQL，找出历史数据中每天K线数不足288根的日期（不含今天，今天数据仍在采集中）：

```sql
SELECT DATE(timestamp) AS dt, COUNT(*) AS cnt
FROM t_kline
WHERE symbol = 'XAUUSDT'
  AND `interval` = '5m'
  AND DATE(timestamp) < CURDATE()
GROUP BY DATE(timestamp)
HAVING COUNT(*) < 288
ORDER BY dt;
```

通过Bash执行：
```bash
mysql -u root -p12345678 aug -e "SELECT DATE(timestamp) AS dt, COUNT(*) AS cnt FROM t_kline WHERE symbol='XAUUSDT' AND \`interval\`='5m' AND DATE(timestamp) < CURDATE() GROUP BY DATE(timestamp) HAVING COUNT(*) < 288 ORDER BY dt;"
```

同时检查是否存在脏数据（时间戳秒数不为0）：
```bash
mysql -u root -p12345678 aug -e "SELECT DATE(timestamp) AS dt, COUNT(*) AS dirty FROM t_kline WHERE symbol='XAUUSDT' AND \`interval\`='5m' AND SECOND(timestamp) != 0 GROUP BY dt ORDER BY dt;"
```

### 第二步：对每个缺口日期进行补齐

对第一步找出的每个缺口日期（格式 YYYY-MM-DD），执行以下操作：

1. **删除该日期现有数据**（避免唯一索引冲突）：
```bash
mysql -u root -p12345678 aug -e "DELETE FROM t_kline WHERE symbol='XAUUSDT' AND \`interval\`='5m' AND DATE(timestamp)='<日期>';"
```

2. **调用OKX补缺接口**（服务需在运行中，端口3131）：
```bash
curl -s "http://localhost:3131/api/historical/okx/xauusdt/custom?startDate=<日期>&endDate=<日期>"
```

### 第三步：验证补齐结果

重新查询确认每天已有288根：
```bash
mysql -u root -p12345678 aug -e "SELECT DATE(timestamp) AS dt, COUNT(*) AS cnt FROM t_kline WHERE symbol='XAUUSDT' AND \`interval\`='5m' AND DATE(timestamp) < CURDATE() GROUP BY DATE(timestamp) HAVING COUNT(*) < 288 ORDER BY dt;"
```

若结果为空，表示所有历史数据已完整。

## 注意事项

- 服务必须在运行中（`mvn spring-boot:run`，端口3131）才能调用OKX接口
- OKX历史K线接口有频率限制，每批次自动等待200ms，通常每天数据约需1-2秒
- 若某天数据超过288根（如289根），也应视为异常并删除重补（查询条件改为 `!= 288`）
- 今天的数据不处理，因为当天K线仍在持续生成中
