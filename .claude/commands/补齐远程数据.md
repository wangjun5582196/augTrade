# 补齐远程服务器K线数据

目标服务器：`43.103.54.183`（MySQL 3306，应用 3131）

按每天288根（5分钟K线，24h×12=288）为标准，自动检测远程服务器的缺口并通过其 OKX 接口补齐。

## 执行步骤

### 第一步：查询缺口日期

```bash
mysql -h 43.103.54.183 -P 3306 -u root -p12345678 aug 2>/dev/null -e "SELECT DATE(timestamp) AS dt, COUNT(*) AS cnt FROM t_kline WHERE symbol='XAUUSDT' AND \`interval\`='5m' AND DATE(timestamp) < CURDATE() GROUP BY DATE(timestamp) HAVING COUNT(*) != 288 ORDER BY dt;"
```

同时检查脏数据（时间戳秒数不为0）：
```bash
mysql -h 43.103.54.183 -P 3306 -u root -p12345678 aug 2>/dev/null -e "SELECT DATE(timestamp) AS dt, COUNT(*) AS dirty FROM t_kline WHERE symbol='XAUUSDT' AND \`interval\`='5m' AND SECOND(timestamp) != 0 GROUP BY dt ORDER BY dt;"
```

### 第二步：对每个缺口日期进行补齐

对第一步找出的每个缺口日期（格式 YYYY-MM-DD），执行以下操作：

1. **删除该日期现有数据**：
```bash
mysql -h 43.103.54.183 -P 3306 -u root -p12345678 aug 2>/dev/null -e "DELETE FROM t_kline WHERE symbol='XAUUSDT' AND \`interval\`='5m' AND DATE(timestamp)='<日期>';"
```

2. **调用远程服务器的 OKX 补缺接口**（让服务器自己从 OKX 拉数据写入自己的 DB）：
```bash
curl -s "http://43.103.54.183:3131/api/historical/okx/xauusdt/custom?startDate=<日期>&endDate=<日期>"
```

### 第三步：验证补齐结果

```bash
mysql -h 43.103.54.183 -P 3306 -u root -p12345678 aug 2>/dev/null -e "SELECT DATE(timestamp) AS dt, COUNT(*) AS cnt FROM t_kline WHERE symbol='XAUUSDT' AND \`interval\`='5m' AND DATE(timestamp) < CURDATE() GROUP BY DATE(timestamp) HAVING COUNT(*) != 288 ORDER BY dt;"
```

若结果为空，表示远程服务器所有历史数据已完整。

## 注意事项

- 远程服务必须在运行中（端口 3131）才能调用 OKX 接口
- 周末及节假日（黄金市场休市）的"缺口"是正常现象，不需要补齐
- 今天的数据不处理，因为当天K线仍在持续生成中
