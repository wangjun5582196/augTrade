## 🔍 根本原因

移动止损参数过于激进：

| 参数 | 旧值 | 问题 |
|------|------|------|
| trigger-profit | 6.0 | 盈利$6-8就启动移动止损，太早！|
| distance | 4.0 | 止损距离太近，容易被扫 |
| lock-profit-percent | 40% | 锁定利润太少 |

**计算示例**（旧参数）：
```
开仓价: $4780.10
盈利: $8.00
锁定40%利润: $8 × 40% = $3.20
新止损价: $4780.10 + $3.20/10 = $4780.42

问题：止损价$4780.42与入场价$4780.10只差$0.32！
价格稍微回调$0.10就触发止损！
```

## ✅ 修复方案

调整移动止损参数，给予价格更多回调空间：

| 参数 | 旧值 | 新值 | 改进 |
|------|------|------|------|
| trigger-profit | 6.0 | **20.0** | 盈利$20才启动，更稳健 |
| distance | 4.0 | **8.0** | 止损距离加倍，减少误触 |
| lock-profit-percent | 40% | **50%** | 锁定一半利润 |

**计算示例**（新参数）：
```
开仓价: $4780.10
盈利: $20.00（触发阈值）
锁定50%利润: $20 × 50% = $10.00
新止损价: $4780.10 + $10.00/10 = $4781.10

改进：止损价$4781.10与入场价$4780.10相差$1.00
有足够空间应对正常波动，且已锁定$10利润
```

## 🎯 移动止损逻辑

```
1. 持仓盈利 < $20  → 使用固定止损（不启动移动止损）
2. 持仓盈利 ≥ $20  → 启动移动止损
   - 锁定50%利润（$10）
   - 止损价 = 入场价 + $1.00
   
3. 价格继续上涨  → 止损价跟随上涨
   - 新止损价 = 当前价 - $8.00
   - 只在新止损价更优时更新
   
4. 价格回调$8+  → 触发移动止损
   - 保护已锁定的利润
```

## 📝 经验教训

1. **移动止损不是越早越好**
   - 过早启动会把止损价拉到入场价附近
   - 正常价格波动就会触发止损
   - 小盈利变成小亏损

2. **必须留足回调空间**
   - 黄金5分钟波动通常$2-5
   - distance=4太小，容易被扫
   - distance=8更合理

3. **触发阈值要合理**
   - trigger=6太低，盈利$6-8就锁定
   - trigger=20更稳健，确保有足够利润

4. **测试参数要用真实场景**
   - 不能只看理论计算
   - 必须考虑市场正常波动
   - 回测数据验证效果
