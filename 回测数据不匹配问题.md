# ⚠️ 回测数据不匹配问题

## 🔍 问题诊断

你选择的回测参数：
- **交易对**：XAUTUSDT
- **周期**：5m
- **时间**：2025年6月1日 - 12月31日

但很可能数据库中的数据不匹配：
- **symbol可能是**：XAUUSD（不是XAUTUSDT）
- **interval可能是**：5（不是5m）
- **时间**：2025年的数据可能不存在（现在是2026年，Bybit免费API可能只提供最近几个月的数据）

## 🎯 解决方案

### 第1步：检查数据库中的实际数据

执行以下SQL查询：

```sql
-- 查看有哪些交易对
SELECT DISTINCT symbol FROM t_kline;

-- 查看有哪些周期
SELECT DISTINCT `interval` FROM t_kline;

-- 查看时间范围
SELECT 
    symbol,
    `interval`,
    MIN(timestamp) as 最早时间,
    MAX(timestamp) as 最晚时间,
    COUNT(*) as 数据量
FROM t_kline 
GROUP BY symbol, `interval`;
```

### 第2步：根据查询结果调整回测参数

#### 情况A：如果数据库显示
```
symbol = XAUUSD
interval = 5
时间范围 = 2026-01-02 至 2026-01-09
```

则回测时应该填写：
- 交易对：**XAUUSD**（不是XAUTUSDT）
- K线周期：**5m**（前端会自动处理）
- 开始时间：**2026-01-02**
- 结束时间：**2026-01-09**

#### 情况B：如果数据库是空的

需要先获取数据：
```bash
# 获取最近7天数据
curl -X POST "http://localhost:3131/api/data/fetch-recent-gold"

# 或获取最近30天数据
curl -X POST "http://localhost:3131/api/data/fetch-month-gold"
```

然后查询数据库确认symbol和时间范围。

### 第3步：关于2025年数据

**重要提示：** 2025年6月-12月的数据已经是7个月前的历史数据！

Bybit免费API通常只提供最近几个月的数据，可能无法获取那么早的历史数据。

**建议：**
1. 使用最近30天的数据进行回测
2. 这样更能反映当前市场状况
3. 数据更容易获取

## 🔍 快速验证

在数据库中执行：
```sql
-- 查看XAUUSD的数据
SELECT COUNT(*) FROM t_kline 
WHERE symbol = 'XAUUSD' AND `interval` = '5';

-- 查看XAUTUSDT的数据
SELECT COUNT(*) FROM t_kline 
WHERE symbol = 'XAUTUSDT' AND `interval` = '5m';

-- 查看所有数据
SELECT 
    symbol,
    `interval`,
    COUNT(*) as count,
    MIN(timestamp) as earliest,
    MAX(timestamp) as latest
FROM t_kline 
GROUP BY symbol, `interval`;
```

2. 在回测页面使用这些实际值
3. 开始回测

   ```bash

   ```

   ```sql

   ```
3. 使用查询到的实际值进行回测
