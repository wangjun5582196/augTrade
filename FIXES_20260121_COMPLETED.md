```yaml

# 修改后

  risk:
    atr-stop-loss-multiplier: 3.0      # 🔥 从2.0提高到3.0
    atr-take-profit-multiplier: 4.0    # 🔥 从2.0提高到4.0
    atr-max-threshold: 6.0             # 🔥 从8.0降到6.0
    
    trailing-stop:
      trigger-profit: 8.0              # 🔥 从10.0降到8.0
      distance: 6.0                    # 🔥 从8.0降到6.0
      lock-profit-percent: 50.0        # 🔥 从60%降到50%
```

**修改原因**：
1. 止损距离2.0倍ATR太紧，导致22笔盈利单被提前止损
2. 止盈距离2.0倍ATR太近，无法让利润充分奔跑
3. ATR>8时止损经常被突破，需要更早停止交易
4. 移动止损触发点10美元太高，需要更早保护利润

    return Signal.HOLD;

**效果预期**：避免暴涨暴跌期间的逆势亏损（如订单#247、#248的$185亏损）

#### 修改5：禁用强趋势逆向交易

```java
// 计算EMA判断趋势方向
BigDecimal ema20 = indicatorService.calculateEMA(klines, 20);
BigDecimal ema50 = indicatorService.calculateEMA(klines, 50);
boolean isUptrend = ema20.compareTo(ema50) > 0;
boolean isDowntrend = ema20.compareTo(ema50) < 0;

// 🔥 P0修复4：强趋势中禁用逆向信号
if (adx.compareTo(new BigDecimal("25")) > 0) {
    if (isUptrend) {
        sellScore = 0;  // 清零SELL评分
        log.info("🔥 强趋势上涨（ADX={}），禁用SELL信号", adx.doubleValue());
    }
    if (isDowntrend) {
        buyScore = 0;  // 清零BUY评分
        log.info("🔥 强趋势下跌（ADX={}），禁用BUY信号", adx.doubleValue());
    }
}
```

**效果预期**：避免强趋势逆向交易的$293亏损（如订单#257的$108亏损）
