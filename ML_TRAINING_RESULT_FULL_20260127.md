## 💡 为什么准确率下降？

### 原因分析

**1. 数据时间跨度更长，市场环境更多样**

```
6个月: 2025-07-28 ~ 2026-01-27 (夏秋冬)
1年:   2025-04-04 ~ 2026-01-27 (春夏秋冬)
```

- 1年数据包含更多市场环境
- 春季市场行为可能与冬季不同
- 模型需要学习更复杂的模式

**2. 测试集时间更近，市场可能变化**

```
6个月测试集: 2025-12-15 ~ 2026-01-27
1年测试集:   2025-11-22 ~ 2026-01-27
```

- 1年模型的测试集包含更早期数据
- 可能遇到不同的市场状态

**3. 模型变得更激进**

- 更倾向于预测涨跌而非震荡
- 虽然召回率提升，但精确率降低
- 产生更多假阳性

## 🎯 实际应用策略调整

基于1年模型的特点，推荐使用策略：

### 策略：超保守震荡过滤器

```java
// 利用1年模型的高震荡精确率（94%）

if (traditionalSignal.isBuy()) {
    MLPrediction mlPred = mlService.predict(context);
    
    // 场景1: ML高置信度预测震荡（精确率94%！）
    if (mlPred.getProbHold() > 0.80) {
        log.warn("❌ ML识别为震荡期（置信度{:.2f}，精确率94%），拒绝", 
                 mlPred.getProbHold());
        return createHoldSignal("ML高置信度识别为震荡");
    }
    
    // 场景2: ML预测上涨，但精确率只有14%，需要更高阈值
    if (mlPred.getProbUp() > 0.75) {  // 提高到0.75
        log.info("✅ ML高置信度支持做多", mlPred.getProbUp());
        return traditionalSignal;
    }
    
    // 场景3: ML预测下跌，也需要高阈值
    if (mlPred.getProbDown() > 0.70) {
        log.warn("⚠️ ML预测下跌，拒绝做多");
        return createHoldSignal("ML预测下跌");
    }
    
    // 场景4: ML不确定，看震荡概率
    if (mlPred.getProbHold() > 0.60) {
        log.info("⚠️ ML倾向震荡，降低信号强度");
        traditionalSignal.setStrength(traditionalSignal.getStrength() * 0.7);
    }
    
    return traditionalSignal;
}
```

**关键调整**：

1. ✅ **主要利用震荡识别**（精确率94%）
2. ⚠️ **涨跌预测要求更高置信度**（0.75+）
3. ✅ **多层次判断**（0.6 / 0.7 / 0.8）
