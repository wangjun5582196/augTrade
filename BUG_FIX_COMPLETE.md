### ✅ Bug #2：持仓保护if条件顺序错误

**问题：**
```java
// 错误代码
if (unrealizedPnL > $10) {
    minHoldingTime = 600;
} else if (unrealizedPnL > $40) {  // 永远不会执行
    minHoldingTime = 900;
}
```

当盈利$40时，先匹配第一个条件（>$10），永远进不了else if。

**修复内容：**
```java
// 正确代码 - 先判断大的，再判断小的
if (unrealizedPnL > $40) {
    minHoldingTime = 900;  // 15分钟
} else if (unrealizedPnL > $10) {
    minHoldingTime = 600;  // 10分钟
} else {
    minHoldingTime = 300;  // 5分钟
}
```

**代码位置：** `TradingScheduler.java` 第190-210行

**预期效果：**
- 盈利>$40的单子持仓15分钟（之前只有10分钟）
- 更多机会触及止盈$80
- 减少提前平仓
