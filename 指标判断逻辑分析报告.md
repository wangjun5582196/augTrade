## 🔍 问题1: 动量指标判断过于宽松

### 当前逻辑
```java
// MomentumCalculator.java
isStrongUp(momentum2 > 0 && momentum5 > 0)
isStrongDown(momentum2 < 0 && momentum5 < 0)
```

### 问题描述
- **过于简单**: 只要价格比2根和5根K线前高，就判定为"强劲上涨"
- **没有阈值**: 哪怕只涨$0.01也算"强劲"，容易产生假信号
- **5分钟K线**: 2根=10分钟，5根=25分钟，窗口太短，噪音较大

### 风险
在震荡行情中，价格上下波动可能频繁触发"强劲"信号，导致：
- ❌ 虚假做多/做空信号增加
- ❌ 在横盘时频繁开仓
- ❌ 信号质量下降

### 建议优化
```java
// 方案1: 增加百分比阈值
private static final double MOMENTUM_THRESHOLD = 0.002; // 0.2%

isStrongUp = momentum2.doubleValue() > price2.doubleValue() * MOMENTUM_THRESHOLD &&
             momentum5.doubleValue() > price5.doubleValue() * MOMENTUM_THRESHOLD;

// 方案2: 使用动量比率代替绝对值
double momentum2Ratio = momentum2.divide(price2, 4, RoundingMode.HALF_UP).doubleValue();
double momentum5Ratio = momentum5.divide(price5, 4, RoundingMode.HALF_UP).doubleValue();

isStrongUp = momentum2Ratio > 0.002 && momentum5Ratio > 0.002;  // 0.2%以上才算强劲

// 方案3: 加速度判断（更严格）
// 不仅要上涨，还要加速上涨
boolean isAccelerating = momentum2.doubleValue() > momentum5.doubleValue() / 2.5;
isStrongUp = momentum2 > 0 && momentum5 > 0 && isAccelerating;
```

## 🔍 问题2: 摆动点判断存在时间延迟

### 当前逻辑
```java
// SwingPointCalculator.java
// 需要左右各5根K线确认摆动点
for (int i = lookback; i < Math.min(klines.size() - lookback, 50); i++) {
    // 从第5根开始查找，最多查找到第50根
}
```

### 问题描述
- **严重滞后**: 摆动高点至少是5根K线（25分钟）之前的
- **突破信号延迟**: 当价格突破摆动高点时，这个高点可能已经是30-60分钟前的了
- **实时性差**: 在快速行情中，等摆动点确认后再突破，机会可能已经错过

### 示例场景
```
时间轴: T0 → T5 → T10 → T15 → T20 → T25 (当前)
价格:   100 → 105 → 103 → 104 → 106 → 108

T10时刻形成摆动高点105（左右各5根确认）
T25当前价格108 > 105，触发"突破摆动高点"信号

但实际上：
- 摆动高点105是15分钟前的价格
- 当前已经涨了3美元(2.9%)
- 此时追高风险较大
```

### 风险
- ❌ 在快速上涨后才发出做多信号（追高）
- ❌ 突破信号的时效性差
- ❌ 7分权重太高，但信号滞后

### 建议优化
```java
// 方案1: 缩短lookback周期（降低延迟）
private static final int DEFAULT_LOOKBACK = 3;  // 从5改为3

// 方案2: 突破信号加入时间过滤
if (lastSwingHigh != null) {
    long minutesSinceSwing = ChronoUnit.MINUTES.between(
        lastSwingHigh.getTime(), 
        klines.get(0).getTimestamp()
    );
    
    // 只有在摆动点形成后15分钟内突破才算有效
    if (minutesSinceSwing <= 15 && currentPrice.compareTo(lastSwingHigh.getPrice()) > 0) {
        isBreakingHigh = true;
    }
}

// 方案3: 突破幅度要求
if (lastSwingHigh != null) {
    double breakoutPercent = currentPrice.subtract(lastSwingHigh.getPrice())
                            .divide(lastSwingHigh.getPrice(), 4, RoundingMode.HALF_UP)
                            .doubleValue();
    
    // 需要突破0.3%以上才算有效突破（避免微小突破）
    isBreakingHigh = breakoutPercent > 0.003;
}

// 方案4: 降低权重
// 考虑到信号延迟，摆动点权重从7分降至5分
```

## 🔍 问题3: 成交量突破依赖动量方向

### 当前逻辑
```java
// CompositeStrategy.java - calculateNewIndicatorsScore()
if (volume != null && volume.isBreakout()) {
    MomentumCalculator.MomentumResult momentum = context.getIndicator("Momentum");
    if (momentum != null) {
        if (momentum.getMomentum2().compareTo(BigDecimal.ZERO) > 0) {
            buyScore += 5;  // 放量上涨
        } else if (momentum.getMomentum2().compareTo(BigDecimal.ZERO) < 0) {
            sellScore += 5;  // 放量下跌
        }
    }
}
```

### 问题描述
- **依赖顺序**: 成交量评分依赖于动量指标的计算结果
- **单一判断**: 只用momentum2（2根K线，10分钟）判断方向，窗口太短
- **缺少价格确认**: 没有考虑当前K线的涨跌情况

### 风险
- 如果动量计算失败，成交量信号就失效
- 可能出现放量但方向判断错误的情况

### 建议优化
```java
// 方案1: 使用当前K线的涨跌判断方向（更实时）
Kline currentKline = context.getKlines().get(0);
BigDecimal klineChange = currentKline.getClosePrice().subtract(currentKline.getOpenPrice());

if (volume != null && volume.isBreakout()) {
    if (klineChange.compareTo(BigDecimal.ZERO) > 0) {
        buyScore += 5;  // 放量阳线
        buyReasons.add(String.format("放量上涨(5分,比率%.2f)", volume.getVolumeRatio()));
    } else if (klineChange.compareTo(BigDecimal.ZERO) < 0) {
        sellScore += 5;  // 放量阴线
        sellReasons.add(String.format("放量下跌(5分,比率%.2f)", volume.getVolumeRatio()));
    }
}

// 方案2: 结合动量和K线，双重确认
if (volume != null && volume.isBreakout()) {
    boolean momentumUp = momentum != null && momentum.getMomentum2().compareTo(BigDecimal.ZERO) > 0;
    boolean klineUp = klineChange.compareTo(BigDecimal.ZERO) > 0;
    
    if (momentumUp && klineUp) {
        buyScore += 5;  // 动量+K线双确认
    } else if (!momentumUp && !klineUp) {
        sellScore += 5;
    }
    // 如果方向不一致，不评分（避免混乱信号）
}
```

## 🔍 问题4: HMA权重过低且逻辑不完整

### 当前逻辑
```java
// CompositeStrategy.java
if ("UP".equals(hma.getTrend()) && hma.isPriceAboveHMA() && buyScore > 0) {
    buyScore += 2;  // 只有2分
```

### 问题描述
- **条件苛刻**: 要求`buyScore > 0`，即必须其他指标已经给出信号才加分
- **权重过低**: HMA是快速趋势指标，2分权重太低，几乎不影响决策
- **逻辑矛盾**: HMA设计为"不直接评分，用于过滤和确认"，但实际又加了2分

### 建议优化
```java
// 方案1: 提高权重，独立评分
if ("UP".equals(hma.getTrend()) && hma.isPriceAboveHMA()) {
    buyScore += 3;  // 提高到3分
    buyReasons.add("HMA上涨趋势(3分)");
}

// 方案2: 作为过滤器而非评分器（更符合设计初衷）
// 在ADX过滤后增加HMA过滤
if (buyScore >= SIGNAL_THRESHOLD && buyScore > sellScore) {
    HMACalculator.HMAResult hma = context.getIndicator("HMA");
    if (hma != null && "DOWN".equals(hma.getTrend())) {
        log.warn("[{}] ⚠️ HMA下跌趋势，做多信号被否决", STRATEGY_NAME);
        return createHoldSignal("HMA趋势相反", buyScore, sellScore);
    }
}

// 方案3: 使用HMA斜率强度加权
if ("UP".equals(hma.getTrend()) && hma.isPriceAboveHMA()) {
    double slopeAbs = Math.abs(hma.getSlope());
    int hmaScore = slopeAbs > 0.001 ? 4 : 2;  // 斜率大的给4分，小的给2分
    buyScore += hmaScore;
    buyReasons.add(String.format("HMA上涨趋势(%d分,斜率%.4f)", hmaScore, hma.getSlope()));
}
```

## 🔍 问题5: 缺少指标之间的冲突检测

### 当前逻辑
```java

buyScore += 5;  // 成交量
buyScore += 7;  // 摆动点

// 最多18分做多信号
```

### 问题描述
- **缺少冲突检测**: 如果动量向上但HMA向下，没有冲突处理
- **没有一致性奖励**: 4个指标都指向同一方向时，应该有额外加分
- **信号质量未分级**: 2个指标vs4个指标的信号，质量差异大但评分方式相同

### 建议优化
```java
// 方案1: 增加一致性检查
int indicatorCount = 0;  // 有信号的指标数量
int bullishCount = 0;    // 看涨的指标数量
int bearishCount = 0;    // 看跌的指标数量

// 统计各指标方向
if (momentum != null && momentum.isStrongUp()) bullishCount++;
if (momentum != null && momentum.isStrongDown()) bearishCount++;
// ... 统计其他指标

indicatorCount = bullishCount + bearishCount;

// 冲突检测
if (bullishCount > 0 && bearishCount > 0) {
    log.warn("[{}] ⚠️ 新指标信号冲突: {}个看涨 vs {}个看跌", 
            STRATEGY_NAME, bullishCount, bearishCount);
    // 降低评分或不评分
    return IndicatorScoreResult.builder()
            .buyScore(0)
            .sellScore(0)
            .build();
}

// 一致性奖励
if (indicatorCount >= 3 && (bullishCount == indicatorCount || bearishCount == indicatorCount)) {
    // 3个以上指标一致，额外+3分
    if (bullishCount == indicatorCount) {
        buyScore += 3;
        buyReasons.add("指标高度一致(+3分)");
    } else {
        sellScore += 3;
        sellReasons.add("指标高度一致(+3分)");
    }
}
```

| 成交量(Volume) | 5分 | 5分 | 放量(1.5x) + 动量方向 |

| **最大总分** | **18分** | **11分** | 所有条件满足 |

### 问题分析

#### 1. 做多/做空权重不对称
- **做多最多18分**: 动量4 + 成交量5 + 摆动点7 + HMA 2
- **做空最多11分**: 动量4 + 成交量5 + HMA 2（没有摆动点）
- **问题**: 做空信号难以达到12分阈值，可能导致做空机会减少

#### 2. 摆动点权重过大且单一方向
- 摆动点突破给7分，占做多总分的39%
- 但没有对称的做空信号（突破摆动低点）
- 如果摆动点判断失误，整个策略偏差较大

#### 3. 与原有策略权重对比
| 原策略 | 权重 |
|--------|------|
| TrendFilter | 12分 |
| Supertrend | 8分 |
| VWAP | 5分 |

**新指标18分 vs 单个最强策略12分**，新指标影响力很大！
