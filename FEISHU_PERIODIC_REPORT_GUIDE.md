# 飞书定期报告功能指南
**更新时间**: 2026-01-09 10:59  
**功能状态**: ✅ 已实现

---

## 📋 功能说明

系统会定期向飞书群发送交易报告，包含：
1. 📊 当前持仓情况（如果有持仓）
2. 📈 今日交易统计数据

---

## ⏰ 执行频率

### 当前配置：每30分钟一次

```java
@Scheduled(cron = "0 */30 * * * ?")
```

**执行时间**：
```
10:00, 10:30, 11:00, 11:30, 12:00, 12:30...
每小时的0分和30分整点执行
```

### 可选频率配置

根据需求可以调整执行频率：

#### 1. 每15分钟（更频繁，更实时）
```java
@Scheduled(cron = "0 */15 * * * ?")
// 执行时间: 10:00, 10:15, 10:30, 10:45...
```

#### 2. 每小时（标准频率）
```java
@Scheduled(cron = "0 0 * * * ?")
// 执行时间: 10:00, 11:00, 12:00...
```

#### 3. 特定时间点（精准控制）
```java
@Scheduled(cron = "0 0 9,12,15,18,21 * * ?")
// 执行时间: 09:00, 12:00, 15:00, 18:00, 21:00
```

#### 4. 工作时间每小时（避免休息时间）
```java
@Scheduled(cron = "0 0 9-21 * * ?")
// 执行时间: 09:00-21:00期间每小时执行
```

## 📊 统计指标说明

### 胜率标识
- ✅ 绿色勾：胜率 ≥ 55%（优秀）
- ⚠️ 黄色警告：45% ≤ 胜率 < 55%（一般）
- ❌ 红色叉：胜率 < 45%（需要改进）

### 盈亏标识
- ✅ 绿色勾：累计盈利（正数）
- ❌ 红色叉：累计亏损（负数）

## 🔍 故障排查

### Q1: 没有收到报告？

**检查清单**：
1. 飞书通知是否启用
```bash
grep "feishu.notification.enabled" src/main/resources/application.yml
```

2. Webhook URL是否配置
```bash
grep "webhook.url" src/main/resources/application.yml
```

3. 系统是否在模拟交易模式
```bash
grep "paper-trading: true" src/main/resources/application.yml
```

4. 查看错误日志
```bash
grep -i "error.*feishu" logs/aug-trade.log | tail -5
```

### Q2: 报告内容不完整？

**检查数据源**：
```bash
# 检查是否有持仓数据
source ~/.bash_profile && mysql -uroot -p12345678 test -e "SELECT * FROM t_position WHERE status='OPEN'"

# 检查今日交易数据
source ~/.bash_profile && mysql -uroot -p12345678 test -e "SELECT COUNT(*) FROM t_trade_order WHERE DATE(create_time)=CURDATE()"
```

### Q3: 想手动触发测试？

**添加测试接口**：
访问 `http://localhost:3131/api/feishu/test-periodic-report`

或在代码中调用：
```java
tradingScheduler.sendPeriodicReport();
```
